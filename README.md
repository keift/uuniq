## About

Unique IDs that will last for thousands of years.

## Features

  * The possibility of collision is impossible
  * Suitable for distributed systems
  * Suitable for sorting and database indexes
  * Snowflake IDs developed by Twitter (X) can be generated
  * Shorter IDs can be generated like YouTube's video IDs

## Usage

Briefly as follows.

> ```javascript
> const Uuniq = require("uuniq");
> const Snowflake = new Uuniq.Snowflake();
> const Symbolic = new Uuniq.Symbolic();
>
> Snowflake.generate(); // "668874142515200"
> Snowflake.generate(); // "668874142515201"
>
> Snowflake.resolve("668874142515200");
> /*
>   {
>     created_at: "2025-03-01T01:31:44.725Z",
>     place_id: 0,
>     sequence: 0
>   }
> */
>
> Symbolic.generate(); // "3iqEsohf_"
> Symbolic.generate(); // "3iqEsohf-"
>
> Symbolic.resolve("3iqEsohf_");
> /*
>   {
>     created_at: "2025-03-01T01:33:47.048Z",
>     place_id: 0,
>     sequence: 0
>   }
> */
> ```

## Documentation

### Constructors

`new Snowflake(options?)`

Generate Snowflake IDs developed by Twitter(X) in 2010. This can generate IDs for approximately 4,463 years according to the specified epoch. Unique IDs can be generated in distributed systems by specifying Place IDs.

> | Parameter | Default | Description |
> | --- | --- | --- |
> | options | | [Object] (optional)<br/>Constructor's options. |
> | options.epoch | `"2025-01-01T00:00:00.000Z"` | [String] \| [Number] \| [Date] (optional)<br/>Date of epoch. |
> | options.place_id | `0` | [Number] (optional)<br/>Place ID for distributed systems. |
>
> returns [String]
>
>
> Example:
>
> ```js
> const Snowflake = new Uuniq.Snowflake({
>   epoch: "2025-01-01T00:00:00.000Z",
>   place_id: 0
> });
> ```

<br/>

`new Symbolic(options?)`

Generate unique IDs like YouTube's video IDs. This can generate IDs for approximately 4,463 years based on the specified period. Unique IDs can be generated by specifying Place IDs in distributed systems.

> | Parameter | Default | Description |
> | --- | --- | --- |
> | options | | [Object] (optional)<br/>Constructor's options. |
> | options.epoch | `"2025-01-01T00:00:00.000Z"` | [String] \| [Number] \| [Date] (optional)<br/>Date of epoch. |
> | options.place_id | `0` | [Number] (optional)<br/>Place ID for distributed systems. |
> | options.charset | `"123456789ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz_-"` | [String] (optional)<br/>Character set of IDs. |
>
> returns [String]
>
>
> Example:
>
> ```js
> const Symbolic = new Uuniq.Symbolic({
>   epoch: "2025-01-01T00:00:00.000Z",
>   place_id: 0,
>   charset: "123456789ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz_-"
> });
> ```
